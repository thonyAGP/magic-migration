# /modules - Module Readiness Dashboard

Display deliverable modules and blockers.

## Usage

```
/modules [--deliverable] [--blockers]
```

## Output

### Deliverable Modules (100% verified)
Subtrees that can be shipped to production and their legacy equivalent decommissioned.

### Close (>80%)
Subtrees almost ready. Shows blocker programs and what status they need to reach.

### Critical Blockers
Programs that if verified would unlock the most modules. Prioritized by impact.

### Global
Total live programs, total modules, delivery percentage.
